Q73 K1
What is a function in C programming?
Solution
A function is a block of code that performs a specific task. It improves modularity, reusability, and readability. Functions may take input (parameters) and return output (return value).
Q74 K1
Differentiate between built-in and user-defined functions.
Solution
  • Built-in functions: Predefined in libraries (e.g., printf(), scanf(), strlen()).
  • User-defined functions: Created by programmers to solve specific problems (e.g., int add(int a, int b)).
Q75 K2
Explain call by value vs call by reference in C with examples.
Solution

Call by Value: The function receives a copy of the variable's value. Changes made inside the function do not affect the original variable.

Example:

void fun(int x) {
    x = 10;
}

Call by Reference: The function receives the memory address of the variable. Changes made using the pointer affect the original variable.

Example:

void fun(int *p) {
    *p = 10;
}
Q76 K3
Write a program to find factorial of a number using recursion.
Solution
#include <stdio.h>
int fact(int n) {
    if (n == 0) return 1;
    return n * fact(n - 1);
}
int main() {
    int num;
    printf("Enter number: ");
    scanf("%d", &num);
    printf("Factorial = %d\n", fact(num));
    return 0;
}
Q77 K3
Write a recursive program to generate Fibonacci series up to n terms.
Solution
#include <stdio.h>
int fib(int n) {
    if (n == 0) return 0;
    if (n == 1) return 1;
    return fib(n - 1) + fib(n - 2);
}
int main() {
    int n;
    printf("Enter terms: ");
    scanf("%d", &n);
    for (int i = 0; i < n; i++) {
        printf("%d ", fib(i));
    }
    return 0;
}
Q78 K2
What is recursion? State advantages and disadvantages.
Solution

Recursion: A function calling itself until a specific base condition is met. It breaks down a problem into smaller, similar sub-problems.

✅ Advantages:

  • Simplifies code for certain problems (e.g., factorial, Fibonacci, tree traversal).
  • Can be more intuitive for recursive problem structures.

❌ Disadvantages:

  • Higher memory consumption due to the call stack.
  • Slower execution compared to iterative solutions due to function call overhead.
  • Risk of a "stack overflow" error if the recursion depth is too large.
Q79 K4
Analyze differences between iterative and recursive approaches.
Solution
  • Iterative: Uses loops (e.g., `for`, `while`) to repeat a block of code. Generally more memory-efficient (O(1) space) and faster.
  • Recursive: Uses function calls to solve a problem. Often simpler to write for certain algorithms but less efficient in terms of memory (O(n) space due to the call stack).

Example: Calculating a factorial. The iterative approach uses a loop and constant memory, while the recursive approach uses the call stack to store intermediate results, consuming memory proportional to the input number.

Q80 K4
Discuss scope, lifetime, and storage classes of functions and variables in C.
Solution
  • Scope: The region of a program where an identifier (variable or function) is accessible. This can be local (within a function) or global (across the entire program).
  • Lifetime: The duration for which a variable exists in memory. A variable's lifetime determines when memory is allocated and deallocated for it.
  • Storage Classes: Keywords that define the scope, lifetime, and memory location of variables.
    • auto: The default for local variables. They have local scope and their lifetime is limited to the function's execution.
    • static: Extends a variable's lifetime to the entire program, even if its scope is local. The value is preserved between function calls.
    • extern: Used to declare a variable that is defined in another file, extending its scope across multiple source files.
    • register: A hint to the compiler to store the variable in a CPU register for faster access.
